第二章 IC和ID卡基本知识

一、IC卡基本介绍

1、IC卡和ID卡判断

把卡放在低频区读ID/HID 或放在高频区读卡类型，哪个能读出来就是哪个型号。

注意，卡片外形和型号无关，相同的芯片可以封装成完全不同的外形：

ID和IC是2种不同型号的卡不可以相互替用，各有各的复制卡。

2、IC卡介绍

（1）M1卡介绍

M1 S50 是国内最常用的卡，也就是我们俗称的 IC 卡。
由飞利浦公司旗下恩智浦（NXP）开发，国产也有兼容卡，但是卡片信息是判断不出来的，卡片信息如下：UID:后面带表是卡号。卡的 0 块是厂家出厂内置的，包含了厂家和卡片信息。不可以修改，只能读。卡详细对照表设置表见【卡片资料】文件夹和参考旧版指导手册。

（2）M1 UID 卡介绍

1、M1 UID 卡是国人针对 M1 S50 卡特制的变种卡，用起来和 M1 S50 完全一样，只是多了一个功能，就是 0 扇区块的数据可以随意修改。因此 UID 号也可以随意修改，厂家信息也可以随意修改，UID 卡因此得名，如果UID卡读卡类型后会显示是UID卡。
    2、UID 卡修改 0 扇区 0 块数据是靠指令进入工厂模式，可以直接对全卡任何数据编辑，不需要密码即可读写卡，写过的卡不用清除也可以直接重复写，同时不怕写坏卡，即使写错 0 块，写坏扇区控制字，也可以随时修复回来，丝毫不影响后续使用，所以应用广泛，深得人心。
    优点是：UID的卡可以完美复制 M1 S50 卡，做到 UID 号也完全一样。读卡机如果有验证 UID 号就没有问题了。

缺点是：现在新的读卡系统，通过检测卡片对特殊指令的回应，可以检测出UID 卡，因此可以来拒绝 UID 卡的访问，来达到屏蔽复制卡的功能。哪就需要换卡了。

（3）M1  FUID、CUID、UFUID 、M1/S50卡介绍

1、FUID卡也是一款支持UID号擦写使用的UID卡，但FUID卡一旦写入备份的dump文件便固化了，丧失了UID卡的特点，起防屏蔽作用，专门对付物业升级的防复卡和防复读头，是物业升级后UID卡不能使用的解决卡。   

2、CUID卡是一款支持重复擦写使用的UID卡，重新写入需要清卡！一般都可以替代普通防屏蔽卡（FUID）。无需锁卡自动起防屏蔽作用，可重复擦写使用无废卡，不会像FUID锁死后才起到防屏蔽作用，更不会因为锁卡而变成一次性卡，。UFUID卡用UID按钮写卡后当UID卡用，也可以锁卡后替换FUID卡使用。

3、M1/S50卡就是普通的IC 卡0块不可改动的，也可以分析数据后直接用m1卡来复制这样任何防火墙都无法检测屏蔽。

注：几种卡使用顺序:UID----CUID----UFUID----FUID-----M1

(复制卡如果刷卡没有反应就代表有小区有防火墙，必须按照此顺序来执行，进行换卡复制。)

（4）简要介绍：

1. M1卡：  普通ic卡（S50），0区0块出厂固化不可以修改，其他扇区可反复擦写，常用物业对应系统发卡使用，不是复制卡，可理解为授权卡。

2. UID卡：  普通复制卡，可以重复擦写所有扇区，遇到防火墙会失效。

3. CUID卡： 升级复制卡，刷卡无反应可用CUID代替UID，70%通过率。可以重复使用。（重复使用需要清卡）

4. FUID卡： 高级复制卡，CUID复制没用的情况下使用，写入成功后0扇0块会固化起防屏蔽作用，变成M1卡。（参考M1）

5. UFUID卡：高级复制卡，是UID和FUID的合成卡，需要锁卡操作，锁卡之后变M1卡，不锁卡就是UID卡。

6.GUID卡：是龙达17年独家针对梯控滚动码研发的，经过几年升级现在已经升级到GUID2代，做滚动码成功率99%

二、滚动码技术

（1）滚动码技术是如何出现

最开始各大公司的卡片内数据都是以明码存放，例如楼层，户号，车位号，手机号，电梯到期时间等，由于卡内的到期时间是明码，用户直接可以修改，后来厂家发现这种漏洞以后，将卡内的这些信息加密处理，但在大神的面前，加密方法也被破解了，再此以后厂家发明了滚动码技术，判断是否为滚动码梯控其实很简单，某些大厂家的梯控都有自己的特征，直接就能看出来，如KD，HY，YH等，不知名的小厂梯控系统通过对比同一张卡两次刷卡的记录，也可以判断出来。

（2）常见的滚动方式

常见的滚动方式有校验从小数到大数，每次刷卡加一，或者从大数到小数每次减少，这种滚动方式常在次数卡上使用，还有公式滚动法，读取卡内的数据与校验，经过读头单片机计算以后，写入新的数据与校验，不同厂家的公式算法不一样。有些厂家的滚动还会以刷卡时间和公式进行滚动，但大同小异。

（3）滚动码的破解方法

1.通过多份数据对比，或是进套数据的方式，找出对应的规则和算法，这种可以完美的对电梯卡进行延期等操作。

2.漏洞改法，通过更改控制位或者修改卡内某些地方的数据来压制卡内数据的滚动，或是通过异或法修改对应值但校验不变方式。

3.初始码数据还原法，这种方法也属于漏洞改法的一种，利用了有些梯控厂家控制检测系统当识别到从未滚动过的卡（就是物业刚办的新卡）数据不会校验上次的逻辑漏洞，让卡内数据永完都是初始数据来对电梯读头进行欺骗，（终极版利用GUID滚动码专用卡可以实现还原法）

4.发卡器发卡，对于一些贼牛逼的厂家的算法无法破解，同时又采用由软件计算和发卡器内单片机双重计算的方式，针对这样的情况，我们使用同厂家同型号同版本的发卡器，通过修改发卡器发卡规则使得规则与物业的发卡规则相同来发卡。

5.经过几年的发展90%滚动码基都可以破解，遇到滚动码不要急，先读出数据用群里常见的软件识别出对应的系统，然后再找对应的资料进行修改，或是在群里请教大家。常见的普通的系统滚动码群里都有相关资料或是软件都可以下载自动参考，众品牌数据原理不相同、处理方法相对也不同，目前最少有60%品牌系统的管理卡是不滚动的，可以借管理卡复制，但涵盖知识面广还是需要多次读取数据对比分析找出对应的规律和长时间积累学习，同时群友在群里也分享有类似资源参考。

6.GUID卡使用方法：

A、原卡拿到物业去重新刷下，然后一次不要使用，回来后直接使用设备备份原始数据并复制到cuid或ufuid上。

B、拿原卡刷2-3次，确认原卡正常，再拿复制卡刷2-3次，确认复制卡也能正常使用。（注意：此时原卡和复制卡都不要再使用。）

3、用第一次备份下来的原始数据写进guid卡里，原卡不要再使用，需要复制多少张卡 就写多少张guid卡，复制的guid卡都可以同时使用。

 

三、防火墙

（1）ID类防火墙

复制卡有4305、5200、5577、8268、8310如果复制成功还是不能使用就换为8268/8310就行了

（2）IC类防火墙低级防火墙

复制卡UID复制成功但不能使用或有的能有的不能开，说明有防火墙，换CUID卡测试

注：1、低级防火墙的原理可以阻挡UID卡，高级防火墙可以更换CUID、FUID、UFUID、M1卡来复制。

 

四、复制卡类型

（1）ID卡类型

常用复制卡4305、5200、5577、8268、8310、

（2）IC卡类型

1.IC卡当ID卡用。只有０扇区０块卡号信息（前4字节（8位））其它扇区没有数据常用复制卡UID

2.复制卡UID复制成功但不能使用或有的能有的不能开，说明有防火墙。常用复制卡CUID、FUID、UFUID、UFUID、M1卡来复制。

3.复制卡CUID复制成功能使用一次或是不能使用，请做数据对比，是否有反复制，对复制卡数据改动，这种可以复制卡进行锁卡操作。 UFUID不锁卡类似UID卡使用，常用复制卡FUID、UFUID（固化后能当m1卡使用）不关联卡号可以用M1卡

五、多卡合并或是模拟到手机上。

1、某些情况下需要多张卡合并一张卡，如果是ic卡，在扇区不冲突的情况下，可以合并一张卡，也可以使用多合一卡，合并多张卡更方便。

2、多合一卡使用，写卡时需要按住对应的按钮，读卡时，也要按住对应的按钮。

3、群文件有群友录制的ic卡模拟手机图文教程和视频教程，可到群文件下载查阅。

4、部分读头兼容性不是太好，手机或手环模拟出来可能不能用，可以使用手机卡贴，或是复合卡贴，更方便兼容性强。
